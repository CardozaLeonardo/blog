Modelo de cajas

Si bien este "modelo de caja" no es una propiedad, es algo que considero que todos deben manejar debido a el manejo de dos propiedades, que son "padding" y "margin", las cuales se explicarán también en esta entrada del blog.

En html, todo elemento debe ser considerado como una caja, y, en CSS, se habla de modelo de caja cuando se refiere al diseño de estos elementos.

El modelo de caja es escencialmente que una caja rodea al elemento html; esta caja está consistida "margin" (márgenes), "border" (bordes), "padding" (relleno) y el contenido -nótese que los elementos que son parte de esta caja, que también son propiedades, los puse entre comillas para que se diferencien-.

Esto se utiliza como una lógica para manejar mejor cada elemento html; y, con esto explicado, entramos a una de estas propiedades nuevas

Margin

La propiedad "margin" sirve para crearle un margen al elemento, o sea, crearle un espacio extra entre el borde y los demás elementos.

Esta propiedad es aplicable a cada elemento html, con la forma: "margin: 10px;", por ejemplo. Por supuesto, se pueden utilizar otras unidades de medida. También se puede ser específico, o sea, "margin-top: 5px;", por ejemplo.

Padding

La propiedad "padding", o relleno, permite eso, crear un espacio vacío entre el contenido del elemento y el borde, y tiene casi el mismo comportamiento que la propiedad "margin", a diferencia que con "padding" no se pueden utilizar ciertos valores pre-definidos, como serían "auto" e "inherit", los cuales se verán en otras entradas del blog.
